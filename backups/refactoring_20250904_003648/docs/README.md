## 📝 ドキュメント作成ガイドライン

### ファイル命名規則

- **形式**: `YYYY-MM-DD_機能分類_内容説明.拡張子`
- **例**: `2025-08-15_CLI機能_ExchangeAnalyticsSystem_CLI機能説明書.md`

### 旧ファイル名の追記

ファイル名変更時は、ファイルの先頭に旧ファイル名を追記：

```markdown
**旧ファイル名**: `元のファイル名.md`
```

### フォルダ分類原則

- **機能中心**: 各フォルダの主要機能が名前から分かる
- **日本語統一**: 英語と日本語の混在を避ける
- **目的明確化**: フォルダ内のドキュメントの目的が一目で分かる

### 更新履歴の記録

各ドキュメントの末尾に更新履歴を記録することを推奨します。
